# Investment Note 작성 스킬

> **공통 규칙**: 작업 시작 전 `_shared/common-rules.md`(로컬 실행 시 스킬 폴더 기준 `../_shared/common-rules.md`)를 읽고 전 구간에 적용한다 — fail-fast, 자동진행/사전확인, ASCII 파일명, 드라이브 3점 검증, 신뢰도 표기, 수집기간 기본 2년, 그리고 **판정 근거 검증 원칙(§11)**: 노출 정규화 %(절대치 단독 금지·개수% 함정)·계약구조 레이어(스팟 vs 장기)·노출 크기보다 구성(드래그 자산)·다축 분리·시황 방향 사전 검증(구조 vs 현재 delta)·수치 최신성. 그리고 **태스크 완료 후 스킬 피드백 루프(§12)**: 최종 산출물 전달 직후(태스크 단위 1회) 새로 확인된 소스·URL·우회법, 반복된 실패 패턴, 사용자가 수동으로 고친 부분을 점검한다. 개선점이 있으면 업로드본 SKILL.md를 즉석 수정하고 동일 diff를 SSOT repo(claude-skills)에 반영한다 (repo 미접근 환경이면 diff를 출력해 "다음 Cowork 세션에서 repo 반영 필요"로 flagging). 없으면 "스킬 업데이트 없음" 한 줄로 종료한다.


리서처(RootN)용 종목 투자 노트 + 모델 제작 워크플로우. 한국(DART)·미국(SEC)·
일본(EDINET)·중국(CNINFO) 상장사 모두 지원. Bloomberg 없이 드라이브 리포트 +
공시 + 컨센서스 + 웹/뉴스만으로 작성하는 것을 기본으로 하되, **컨센서스는 3단계
위계**를 따른다: (1) BEst 덤프(`Model/*_bbg_*.xlsx` Consensus/Revisions 시트,
`bbg-sync` 인제스트분) > (2) CapIQ estimates export(`리서치/consensus/`) >
(3) 무료 소스(FnGuide/리포트 Consensus 박스/Kabuyoho/Minkabu). 노트 작성 전
Drive에서 (1)(2)를 먼저 찾고, 있으면 그것을 컨센 기준으로 쓰고 무료 소스는
검증용으로만 쓴다. 어느 단계 소스를 썼는지 노트 요약에 명시한다.

## 0. 기본 설정 (묻지 말고 기본값으로 적용)
아래는 **기본값**이다. 매번 사용자에게 되묻지 말고 바로 적용한다.
사용자가 명시적으로 다른 형식/깊이/언어를 요청할 때만 그에 맞춰 변경한다.
- **산출물 형식: Excel 파일 1개만 생성** (Word/PDF 생략). 첫 시트 '투자요약'에
  노트 핵심을 임베드해 한 파일로 읽히게 한다(§8-6 구조 준수).
- **노트 깊이: 1페이지 요약.** 단어·숫자 위주, 서술은 최소화한다.
  문장형 설명 대신 표·키워드·수치로 압축한다.
- **언어: 국문.**
- (사용자가 Word 풀 노트·영문 등을 명시 요청하면 §5를 추가로 수행한다.)

## 데이터 활용 원칙 (이 스킬의 핵심 — 전 단계에 적용)
1. **숫자·데이터는 가능한 모두 활용한다.** 검색·드라이브·공시·뉴스에서 나오는
   거의 모든 수치를 끌어온다. **신뢰도가 낮은 데이터도 버리지 않고 활용하되,
   해당 항목에 "신뢰도 낮음"을 명시 표기한다**(셀 주석 또는 인접 셀에 텍스트).
2. **세그먼트를 최대한 잘게 나눈다.** growth가 어느 세그먼트에서 나오는지가
   숫자로 분명히 드러나도록, 제품 × 고객 단위까지 분해한다(§8-1).
3. **Back-of-napkin math를 가능한 많이 사용한다.** 특히 **매출을 P(단가)와
   Q(물량)로 분리하는 작업은 가능한 한 반드시 수행한다**(§3, §8-3).
4. **필요 시 뉴스·IR 자료를 추가로 더 검색한다.** 대상 기업뿐 아니라
   **경쟁사, supply chain 내 다른 기업(고객·모회사·소재/장비 협력사)**의
   실적·가이던스·수주·IR 발언까지 포함한다(§8-2).

## 1. 자료 수집 (research first — 출력 스킬 읽기 전에 완료)
1. 종목 식별: 종목명/티커 → 거래소·코드 확인(웹 검색 1회).
2. 구글 드라이브 검색: `search_files`로 `fullText contains '종목명'` 또는 전용 폴더.
   - 증권사 리포트(최신순 우선: 가장 최근 + 가장 심층 deep-dive), 사업보고서/IR.
   - 결과가 토큰 초과로 파일 저장되면 bash `jq`로 제목·id 추출.
2b. **로컬 다운로드 폴더 스캔(데스크톱 연결 시 필수, 재수집 전에 먼저):** 데스크톱
   브리지가 연결돼 있으면 연결된 다운로드/드라이브 폴더(예: `~/Downloads`,
   `Desktop\Claude (drive 연동)\다운로드`, `Desktop\Drive`)를 `device_list_dir`로
   훑어 종목명·티커·경쟁사·"실적/컨센/IR/사업보고서" 키워드가 걸리는 파일(리포트 PDF,
   IR/결산 PDF, xlsx 모델, BEst/CapIQ export)을 찾는다. 있으면 `device_stage_files`로
   staging해 **Drive 자료와 동일하게 재사용**(중복 재수집 금지). 최근 modifiedTime
   우선. 브리지 미연결/폴더 접근불가면 fail-fast(1~2회 시도 후 건너뜀)하고 Drive만으로
   진행하되 "로컬 폴더 미확인"을 한 줄 flagging. 찾은 로컬 파일은 소스로 명시(파일명·경로).
3. 리포트 정독: `read_file_content(fileId)`로 2~4개 핵심 리포트 + 사업보고서.
   - 사업보고서가 크면 저장된 파일을 bash python/grep으로 섹션 추출
     (매출실적·주요제품 단가·생산설비·가동률·원재료·CAPEX·주주현황).
4. 컨센서스(3단계 위계): **먼저 Drive에서 BEst 덤프(`Model/*_bbg_*.xlsx`)와 CapIQ
   export(`리서치/consensus/`)를 찾는다.** 있으면 그것이 컨센 기준(1·2차).
   없을 때 무료 소스: FnGuide(comp.fnguide.com)는 JS 렌더링이라 web_fetch가 비면,
   최신 증권사 리포트의 "Consensus Data" 박스를 인용(= FnGuide 컨센). 네이버금융은 차단됨.
5. **확장 검색(필수 원칙):** 뉴스·IR을 추가로 검색해 데이터를 보강한다.
   대상 기업 + **경쟁사 + supply chain 기업(고객·모회사·소재/부품/장비사)**의
   실적·가이던스·수주·CAPA·IR/컨퍼런스 발언을 모은다(§8-2). 부족하면 사용자에게도 요청.

## 2. 핵심 데이터 추출 (표로 정리, 저신뢰 항목은 "신뢰도 낮음" 표기)
- 과거 손익(연결): 매출/매출총이익/영업이익/세전/지배순익. **최소 최근 3년,
  확보되면 더 길게(5년+).**
- 분기 실적: **최근 3년치 분기**(최소 12분기)의 매출·OP·OPM·지배순익(리포트 분기표).
  더 길게 확보 가능하면 연장.
- 세그먼트 매출: 사업보고서 "매출실적"(제품별, 연결). **최대한 세분화**(§8-1).
- 제품별 단가(원/개)와 매출 → **물량 역산용(P×Q 분해의 입력)**.
- 선행지표: CAPA(억원/물량), 가동률, 수주잔고, 유상증자·증설 공시.
- 컨센서스: **향후 3년** 매출/OP/지배순익(여러 증권사 비교).

## 3. Back-of-napkin 계산 (가능한 모든 것, P×Q는 반드시)
- **물량 Q = 매출 ÷ 단순평균단가 P**(사업보고서). 천개/백만개 단위.
  → 가능한 모든 세그먼트/제품에 대해 **매출 = P × Q 분해를 반드시 수행**한다.
- **성장 분해**: 매출증가율 ≈ (1+물량증가)×(1+단가증가)−1. 물량 vs 단가 기여 분리.
- **가동률 = 세그먼트 매출 ÷ CAPA(매출환산)**. 리포트 가동률 코멘트와 교차검증.
- **영업레버리지**: 두 연도 (매출,OP)로 한계기여마진 CM=ΔOP/Δ매출 산출 →
  내재 고정비 = 매출×CM − OP → 손익분기 = 고정비÷CM.
- **가동률→마진 민감도**: OPM = (매출×CM − 고정비)/매출, 매출수준×CM 표.
- 모든 hardcode 옆에 출처 주석. 저신뢰 입력은 "신뢰도 낮음" 표기.

## 4. Excel 모델 (xlsx 스킬 읽고 openpyxl) — 기본 산출물
시트 구성(§8-6 표준): 투자요약 / 실적추정(연간) / 분기모델 / 세그먼트(세분화) /
단가·물량 / 영업레버리지 / 고객·협력사 / [고객]_영향 / [업사이드]_시나리오 /
시나리오매트릭스 / 컨센vs추정 / 밸류에이션 / 가정·출처.
- **시계열: 연간·분기 모두 작성.** 연간 = 최근 3년 + 향후 3년(≥6개 연도),
  분기 = 최근 3년 + 향후 3년(≥24개 분기). **더 긴 시계열 확보가 가능하면 더 길게.**
- 단위 억원 통일, 폰트 Arial, 색코딩(파랑=입력, 검정=수식, 초록=시트참조).
- 음수 괄호, 0은 "-", % 0.0%, 배수 0.0x.
- **수식 사용**(하드코딩 금지), 시트간 링크로 연결.
- 추정: 세그먼트 합산 → OPM·순익률 가정 적용 → 컨센/증권사와 비교표.
- 분기 합산 = 연간 추정과 일치하도록 입력값 튜닝.
- 밸류: 시나리오(Bear/Base/Bull) 목표 PER×EPS, 멀티플 추이.
- 저신뢰 수치 셀에는 "신뢰도 낮음" 주석/표기를 남긴다.
- 저장 후 **반드시** recalc 스크립트(claude.ai 실행 환경 기준 `/mnt/skills/public/xlsx/scripts/recalc.py`, 환경 따라 경로 탐색)로 재계산·오류 0 확인.
- 주의: 한글 파일명이 sandbox에서 NFC/NFD 문제를 일으키면 ASCII 파일명 사용.

## 5. Word 노트 (옵션 — 사용자가 풀 노트를 명시 요청할 때만)
기본은 Excel 단독이다. 사용자가 Word 풀 노트를 요청하면 docx 스킬 읽고 docx-js로 작성.
구조: 제목/투자의견 배너 → Executive Summary → 스냅샷표 → 요약실적표 →
(1)기업개요·최근이벤트 (2)투자포인트 (3)분기 추이 (4)Back-of-napkin
(단가·물량/가동률/마진민감도) (5)추정·컨센 비교 (6)세그먼트 (7)밸류에이션
(8)리스크 (9)결론 → 출처 → 면책.
- 폰트 Malgun Gothic(국문), US Letter, 표는 DXA 이중 너비.
- 분기표는 전치(행=분기) 레이아웃이 portrait에 적합.
- docx-js 작성 시 멀티바이트 문자(×,→,÷ 등)는 Write 툴 truncation 위험 →
  bash heredoc('EOF')로 생성하거나 x/-> 등으로 대체.

## 6. 검증 (필수)
- Excel recalc 오류 0, 분기↔연간 합산 일치, 성장률·물량(P×Q) 재계산.
- 컨센서스·공시 숫자 교차대조, 내 추정이 컨센 대비 어느 영역인지 명시.
- **§8-10 up/down 검증(FY1·FY2)**: 함의주가 = EPS×목표PER, up/down% = 함의주가/현재가−1 재계산 오류 0; 확률가중 기대주가가 컨센·현재가 back-out과 정합; gap 브릿지의 driver 합이 총 gap과 일치.
- (Word를 만든 경우에만) 노트를 PDF로 변환 후 이미지로 표·레이아웃 육안 확인.

## 7. 전달
- `present_files`로 **Excel 공유**(Word를 만든 경우 함께), 요약은 간결하게.
- 불일치·한계(주식수 변동, 세그먼트 분기분할 추정, 저신뢰 항목 등) 투명하게 고지.
- 분기 자동 업데이트가 유용하면 스케줄링 제안.

## 원칙
- 계산 가능한 것은 back-of-napkin으로 직접 계산한다(P×Q 분해는 반드시).
- 세그먼트는 growth 출처가 드러나도록 최대한 잘게 나눈다.
- 추정은 보수/베이스/낙관을 함께 제시하고, 컨센서스와의 위치를 밝힌다.
- 내부 참고용 면책을 항상 포함한다.

## 8. 고급 패턴 (심화 분석 시 필수 적용)

### 8-1. 세그먼트 최대 세분화
- 합산 라인을 수요 동인별로 끝까지 분해한다. 예: '통신'을 광(光, 데이터센터 고객)과 RF(무선, 기지국 고객)로, 가능하면 제품 x 고객 단위까지.
- 연간뿐 아니라 분기 모델에도 동일 세분화를 반영한다.
- 애널리스트마다 귀속 추정이 다르면(예: 특정 고객향 매출) 단일 숫자 대신 범위로 명시하고 출처를 단다.

### 8-2. 고객 · 협력사 · 경쟁사 데이터 보강
- 핵심 고객 / 모회사 / 경쟁사 / supply chain 협력사의 실적 · 가이던스 · 수주 · IR/컨퍼런스 발언을 뉴스 · SEC/EDGAR · IR · 셀사이드에서 수집한다.
- 별도 '고객·협력사' 시트로 정리하고 항목마다 신뢰도를 표기한다: 별(공시/감사보고), 반(실적콜·리포트 2차인용), 세모(뉴스·추정/검증필요 = "신뢰도 낮음").
- 예: 광 고객(Lumentum)의 분기 실적·제품별 성장률·CAPA·장기목표, 모회사(RFHIC)의 수주·벤더선정 타이밍, 경쟁사(NXP) 철수 등.

### 8-3. 고객 성장 -> 자사 영향 Back-of-napkin (핵심)
- 고객사 발언/가이던스(예: Lumentum이 Mizuho 컨퍼런스·실적콜에서 '펌프레이저 +80%, 공급부족' 언급)를 자사의 수주·매출·영업이익·EPS·목표주가 영향으로 환산한다.
- 연결 고리: 자사 매출 = 고객 핵심부품 물량 x 자사 콘텐츠(패키지 ASP x 수량). 자사 물량은 (자사 매출 / 단순평균단가)로 역산 = P×Q.
- 가정(ASP·환율·증분마진·순익전환율)을 입력 셀로 두고, 고객 성장률 Bear/Base/Bull -> 자사 매출/증분OP/증분EPS/주당가치(목표PER 환산)를 산출한다.
- 고객이 세부(부품 단위) 매출을 공시하지 않으면 그 수치는 '추정·신뢰도 낮음'으로 명시한다.

### 8-4. 업/다운사이드 시나리오 (타이밍·규모·확률)
- 옵션성 강한 성장 동력(예: GaN 무선)은 촉매 x 타이밍 x 확률 표 + Bear/Base/Bull 매출 시나리오 + 확률가중 기대값으로 분석한다.
- 시나리오별 전사 영향(총매출·OP·지배순익·목표주가·현재가 대비)을 모델 다른 시트와 연동(수식)해 보여준다.
- 확률가중 기대값이 베이스 추정과 부합하는지 교차검증한다. 다운사이드(사이클·타이밍·미확정 수주)도 반드시 별도 명시.
- 여기서 도출한 Bear/Base/Bull은 **§8-10에서 FY1·FY2 EPS → 목표가 → 컨센 대비 up/down %**로 환산한다.

### 8-5. 신뢰도 표기 원칙
- 모든 외부 수치에 별/반/세모 신뢰도를 단다. 세모(단일 뉴스·검증필요)는 본문에서 별도 강조(빨강 등)하고 **"신뢰도 낮음" 텍스트를 함께** 적는다.
- "검색으로 나오는 가능한 모든 수치"를 최대한 포함하되, **저신뢰 수치도 버리지 않고** 출처와 신뢰도("신뢰도 낮음")를 항상 함께 적는다.

### 8-6. 단일 Excel 산출 구조 (기본 모드)
- 기본 산출은 Excel 1개다. 첫 시트 '투자요약'에 노트 핵심(투자의견 배너 · Executive Summary · 투자포인트 · 스냅샷 · 요약실적)을 wrap 텍스트/병합셀로 임베드해 **1페이지 요약(단어·숫자 위주)**으로 읽히게 한다.
- 권장 시트 구성(심화): 투자요약 / 실적추정(연간, 3+3년) / 분기모델(3+3년) / 세그먼트(세분화) / 단가·물량(P×Q) / 영업레버리지 / 고객·협력사 / [고객]_영향 / [업사이드]_시나리오 / 시나리오매트릭스(§8-9) / 컨센vs추정(§8-10, FY1·FY2 up/down) / 밸류에이션 / 가정·출처.
- '투자요약' 시트에는 §8-10 결과인 **FY1·FY2 컨센 대비 up/downside %와 한 줄 thesis**를 반드시 노출한다.
- 연간·분기 모두 최근 3년 + 향후 3년을 채우고, 가능하면 더 긴 시계열로 확장한다.

### 8-7. FnGuide 셀사이드 PDF 수집 (보조)
- 올바른 다운로드 엔드포인트는 POST `/Research/GetPdfFile`(구버전 `/ResearchPdf/GetPdfFile` 아님). 인증은 세션 쿠키.

- 계정에 '일일 다운로드 한도'가 있어 대량 수집 시 막힐 수 있다(오류 페이지에 '일일' 문구). 한도 초과 시 무리하게 재시도하지 말고 사용자에게 알리고 익일 재시도를 제안한다.
- 검색/목록 수집은 한도와 무관 -> 한도 초과 시에도 리포트 목록(제목·날짜·rptId)은 제공한다.

### 8-8. 기술적 주의 (산출 안정성)

- Excel은 저장 후 반드시 recalc로 오류 0 확인, 분기<->연간 합산 일치, 시나리오 확률가중<->베이스 정합성 검증.

### 8-9. 시나리오 매트릭스 (commodity·소수 세그먼트 좌우 종목 필수)
실적이 소수 commodity/세그먼트에 좌우되는 종목은 §8-4(확률형 up/down 시나리오)에 더해 **2변수 교차 매트릭스**를 표준 산출한다.
- **swing 변수 2개**를 고른다(예: 핵심 소재/제품 가격 x 물량 또는 peer 선행지표). 각 변수를 2~3 수준(Low/Base/High)으로 놓고 **교차한 OP·EPS 매트릭스**를 만든다.
- 각 셀에 **vs 전년**과 **vs 컨센** 차이를 함께 표기해, 어느 조합이 컨센을 깨는지 한눈에 보이게 한다.
- 별도 '시나리오매트릭스' 시트로 산출하고 모델 입력셀과 수식 연동한다(하드코딩 금지).

| OP (vsYoY·vs컨센) | 가격 Low | 가격 Base | 가격 High |
|---|--:|--:|--:|
| 물량 Low |  |  |  |
| 물량 Base |  |  |  |
| 물량 High |  |  |  |

- **back-out(현재가 함의 역산)**: 현재 주가·목표PER에서 역산해 **시장이 이미 가정한 swing 변수 수준**을 구한다(예: 지금 멀티플이 함의하는 소재가격 회복률·물량). 내 Base 가정과의 gap을 한 줄로 명시 = 진입 마진.

### 8-10. 컨센서스 대비 Up/Downside — Forward 1·2년 (모든 노트 필수)
투자 판단의 핵심은 "컨센 대비 내가 어디서, 왜 다른가"다. Up/downside는 **절대 수준이 아니라 컨센서스 대비 변이(variant view)**로 정의한다 — "컨센이 놓친 것"이 없으면 up도 없다. **FY1(당해)·FY2(차년)를 핵심 축**으로 아래 (a)~(e)를 표준 산출하고, 전용 시트 '컨센vs추정'에 담아 밸류에이션·시나리오매트릭스 시트와 **수식 연동**(하드코딩 금지)한다. 결과 요약(FY1·FY2 up/down %와 한 줄 thesis)은 '투자요약' 시트에 노출한다.

**(a) 컨센 베이스라인 캡처 (FY1·FY2 각각)** — §1-4 3단계 위계 소스 사용
- 매출·OP·지배순익·**EPS**의 컨센 mean, 확보 시 high/low·추정기관 수(N)·**리비전 방향(1M/3M Δ, 상향/하향)**. 컨센 함의 성장률(YoY)도 계산.
- 리비전이 상향/하향 중인지 = 컨센 자체의 모멘텀. 내 뷰가 그 리비전을 유발하는 방향인지 명시.

**(b) 我추정 vs 컨센 gap 브릿지 (FY1·FY2)**
- 라인별(매출/OP/EPS) `내 추정 − 컨센 = 절대·%`. 
- gap을 **driver별로 분해**: 어디서 갈리나(물량 / 단가 / mix / 마진 / 일회성). 각 gap 항목을 (i) 진짜 뷰 차이 vs (ii) 단순 타이밍(분기 귀속) 차이로 태깅 → 이 분해가 곧 variant view의 근거.
- 세그먼트 시트(§8-1)·P×Q(§8-3)와 연동해 gap의 출처 세그먼트를 지목한다.

**(c) 시나리오 → EPS → 목표가 → up/down (FY1·FY2)**
- §8-9 매트릭스·§8-4 확률표에서 Bear/Base/Bull **FY1 EPS·FY2 EPS**를 도출(입력셀 연동).
- exit 멀티플(목표 PER, 과거 밴드·peer 근거) 가정 → 시나리오별 함의주가 = EPS × PER.
- 각 시나리오 **현재가 대비 up/downside %**, 확률가중 기대주가(FY1·FY2 각각). 컨센이 어느 시나리오에 위치하는지 표기(보통 Base±).

| FY | 항목 | Bear | Base | Bull | 확률가중 | 컨센 위치 |
|---|---|--:|--:|--:|--:|---|
| FY1 | EPS |  |  |  |  |  |
| FY1 | 함의주가(×목표PER) |  |  |  |  |  |
| FY1 | vs 현재가 % |  |  |  |  |  |
| FY2 | EPS |  |  |  |  |  |
| FY2 | 함의주가 / vs현재가% |  |  |  |  |  |

**(d) 현재가 back-out (시장 함의 역산)**
- 현재가·fair PER에서 역산 → **시장이 프라이싱한 FY1·FY2 EPS**. 컨센·내 Base와 비교.
- 해석: 시장 함의 EPS < 내 Base면 upside 여지(컨센 상회 시 리레이팅), > 내 Base면 downside 리스크. 상방/하방 **asymmetry(skew)**를 한 줄로 결론.

**(e) 촉매·리비전 경로 (다음 4~8분기)**
- 향후 1~2년 내 컨센을 내 방향으로 움직일 이벤트(수주·가동·단가·CAPA·규제·경쟁구도)를 **FY1 유발분 / FY2 유발분**으로 나눠 타이밍 매핑. 각 촉매에 방향(+/−)·규모(EPS 영향 %)·확률 표기.
- 최종 한 줄 thesis: "컨센이 놓친 것 = ___ → FY1 EPS +__% / FY2 EPS +__% → up/downside __% / __%".

**출력 시트 '컨센vs추정'**: 열 = FY1·FY2, 행 = [컨센 mean·N·리비전 → 我추정 → gap%(driver 분해) → Bear/Base/Bull EPS → 목표PER → 함의주가 → up/down% → 확률가중 → 현재가 back-out]. 저신뢰 컨센(단일 리포트 인용 등)은 "신뢰도 낮음" 표기. 검증: (c)의 확률가중 기대주가가 (a) 컨센·(d) back-out과 정합적인지 교차확인.
