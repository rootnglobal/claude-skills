# 4. 표준 로깅 모듈 명세

> 모든 코드는 이 문서의 로깅 표준을 따른다. console.log, print 등 임시 로그는 절대 사용하지 않는다.

---

## 로그 레벨 정의

| 레벨 | 용도 | 예시 |
|------|------|------|
| **DEBUG** | 개발 중 상세 추적 정보 | 함수 진입/퇴장, 변수값 변화 |
| **INFO** | 정상 동작 확인 이벤트 | 서버 시작, 요청 처리 완료, 사용자 로그인 |
| **WARN** | 잠재적 문제, 당장 에러는 아님 | 응답 시간 임계치 초과, 재시도 발생 |
| **ERROR** | 기능 실패, 복구 가능 | API 호출 실패, 데이터 유효성 검증 실패 |
| **FATAL** | 시스템 중단 수준의 치명적 오류 | DB 연결 불가, 필수 설정 파일 누락 |

## 표준 로그 포맷

```
[YYYY-MM-DD HH:mm:ss.SSS] [LEVEL] [MODULE] [FUNCTION] MESSAGE {context}
```

**출력 예시:**

```
[2026-04-13 14:23:01.456] [ERROR] [auth] [validateToken] 토큰 검증 실패 {userId: 'u123', reason: 'expired', expiredAt: '2026-04-13T12:00:00Z'}
[2026-04-13 14:23:01.789] [INFO] [api] [fetchUser] API 호출 완료 {elapsed: 142ms, statusCode: 200}
[2026-04-13 14:23:02.001] [DEBUG] [parser] [parseInput] 함수 진입 {input: {type: 'csv', rows: 150}}
```

## 로깅 규칙

### 함수 진입/퇴장 로그
- 핵심 비즈니스 함수는 진입 시 `DEBUG`, 퇴장 시 `DEBUG` + 결과를 로깅

### 에러 context 필수
- `ERROR` 이상은 관련 변수값, 입력값, 스택 정보를 `{context}`에 첨부

### 외부 호출 전후 로그
- API 호출, DB 쿼리 등 외부 시스템 호출은 호출 전(`INFO`)과 응답 후(`INFO`/`ERROR`) 로깅

### 민감 정보 마스킹
- 비밀번호, 토큰, 개인정보는 로그에 남기지 않거나 마스킹 처리
- 예: `{password: '****', token: '...abc123'}`

### 성능 측정 로그
- 주요 처리의 소요 시간을 `INFO` 레벨로 기록
- 예: `DB 쿼리 완료 {elapsed: 142ms}`

## 금지 사항

- `console.log()`, `console.error()` 사용 금지
- `print()`, `print('debug')` 사용 금지
- 로그 메시지에 하드코딩된 문자열만 사용 금지 (반드시 context 첨부)
- 배포 전 로그 삭제 금지 (로그는 영구적으로 유지)

## 로깅 프롬프트

```
4logging.md를 읽고, 지금 구현하는 모든 함수에 로깅 표준을 적용해. 
console.log나 print는 절대 사용하지 마. 반드시 로거 모듈을 통해서만 로그를 남겨.
```

```
에러가 발생했어. 로그를 확인하고, [ERROR] 레벨 로그에서 context를 분석해서 원인을 찾아줘.
```
