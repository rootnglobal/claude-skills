# 출력 템플릿 — 섹션별 작성 가이드

Company Primer 본문(§3의 11개 섹션)을 아래 순서·질문·표 양식으로 채운다.
공통 원칙: 전문용어엔 쉬운 설명+비유, "이 제품이 없으면 고객에게 무슨 문제?" 렌즈,
매출≠이익≠성장 분리, 숫자로 driver 판별, 과장 금지, 출처·신뢰도 표기, verdict(점수·매수/매도·목표주가) 금지.

문서 맨 위 **요약 박스**: 회사 한 줄 정의 / Q×P×Margin 한 줄 / 핵심 포인트 3~5개.

## 1. 회사 한 줄 정의
"무엇을 만들어 → 어떻게(특히 반복적으로) 버는가"가 한 문장에.
예: "DEF는 AI 서버용 고다층 기판을 뚫는 초정밀 드릴을 만드는 회사로, 고객 생산량 증가와 기판 난도 상승이 동시에 수혜로 이어진다."

## 2. 무엇을 만드는 회사인가
- 핵심 제품/서비스 / 고객 공정·시스템 안 위치 / 완제품·부품·소재·장비·시공 중 무엇 / 일회성 vs 소모품 / 수명·교체주기 / 성능이 왜 중요한가.
- **반드시 쉬운 비유.** 예: "MFC = 가스량을 조절하는 초정밀 수도꼭지".

## 3. 산업 구조 (5+2 Forces)
각 force를 단순 설명이 아니라 **이 회사에 유리/불리/중립**으로 평가한다.
1. 기존 경쟁 강도  2. 신규 진입 위협  3. 대체재 위협  4. 공급자 교섭력  5. 고객 교섭력
6. **규제/정책/보조금/안보** (수출통제·국산화·보조금 등)  7. **Capex cycle / 고객 투자 timing** (지금 사이클의 어디인가)

산업 매력도 표:

| 항목 | 평가 | 이 회사에 의미 |
|---|:--:|---|
| 산업 성장성 | High/Mid/Low |  |
| 가격결정력 | High/Mid/Low |  |
| 진입장벽 | High/Mid/Low |  |
| 고객 집중 리스크 | High/Mid/Low |  |
| Capex timing 가시성 | High/Mid/Low |  |

## 4. 밸류체인 위치 & 경쟁구도
이 회사가 사슬 어디에 있고 누구 덕에 버는지. Upstream(원재료·장비·부품) / Midstream(가공·소재·모듈) / Downstream(고객·최종 end-market)로 그린 뒤 위치를 찍는다.

| 밸류체인 단계 | 주요 기업 | 병목 여부 | 가격결정력 | 이 회사 노출 |
|---|---|:--:|:--:|---|

- **인증/qualification 기간**(진입장벽), **고객 dual sourcing 가능성**, **고객사 대체 가능성**을 한 단락으로.
- **주요 peer와 이 회사의 차별점**(핵심 노출·포지션 위주)만 짧게. **전체 peer 밸류에이션 비교·랭킹은 `comparative-investment-note`로 위임**(여기서 표 만들지 않음).

## 5. 고객은 왜 이 회사 제품을 사는가 + Moat
- 고객군·end-market / 구매 결정 기준(품질·수율·안정성·납기·인증·가격 중 결정 요인) / 원가 비중 vs **고장·불량 시 손실** / 전환비용(인증기간·수율 리스크·설계 lock-in) / capex→매출 lag.
> 고객은 [문제] 때문에 이 회사 [제품]을 산다. 원가 비중은 [큰/작은]지만 [수율/안전/납기]에 영향을 줘 쉽게 못 바꾼다.

"돈으로 연결되는 moat" 표(기술력 자랑이 아니라 **실적으로 이어지는** 해자):

| Moat 요소 | 강도 | 실적 연결 방식(어떻게 매출·마진을 지키나) |
|---|:--:|---|
| 기술 난도 | High/Mid/Low |  |
| 인증/qualification | High/Mid/Low |  |
| switching cost / 고객 lock-in | High/Mid/Low |  |
| 가격결정력 | High/Mid/Low |  |
| capacity / 조달력 | High/Mid/Low |  |

## 6. 돈을 버는 방식 (매출 발생 구조)
6개 유형 중 분류(복수·비중 표시): ①장비 판매형 ②소모품/부품 반복형 ③프로젝트/시공형 ④소재/화학 공급형 ⑤서비스/유지보수형 ⑥라이선스/IP형.
- 매출은 고객 **capex 연동**인가 **생산량 연동**인가 / 수주→매출 lag / 수주잔고가 future revenue를 얼마나 설명 / 변동성 / recurring 비중 / 소모품·서비스 비중↑ 시 margin↑인가.
- **commodity price pass-through 여부**와 **FX 민감도**(수출 비중·통화)도 한 줄씩.

## 7. 세그먼트 Breakdown (숫자) — 매출·이익·성장의 출처
**driver 판별의 핵심. 서술이 아니라 표(숫자)로.** 세 축 각각 최소 최근 2~3년치(델타 계산에 ≥2개 기간 필요):
**(A) Country/Region, (B) Product/Segment, (C) Customer.**

| 구분 | 매출(금액) | 매출 비중% | YoY% | 이익(OP/GP) | 이익 비중% | margin% | 성장 기여도% | 신뢰도 |
|---|--:|--:|--:|--:|--:|--:|--:|:--:|

- **성장 기여도%** = (당기 세그먼트매출 − 전기 세그먼트매출) ÷ 전기 전사매출 → 합 = 전사 성장률. (또는 세그먼트 Δ매출 ÷ 전사 Δ매출, 합=100%.)
- 부문 OP 공시 없으면 GP·추정 margin 근사 + "신뢰도 낮음". 신뢰도: 별(공시) · 반(콜·리포트) · 세모(뉴스·추정).
- 고객(C축)은 공시 약함 → 고객집중도(Top-N 비중)·핵심고객명·supply-chain 역산(고객 물량×자사 content) + 추정 태그.

**표 아래 driver 판별 1단락(필수):**
> 매출은 [A]가 가장 크지만(X%), 이익은 [B]에서 더 나고(margin Y%), 최근 성장의 Z%는 [C]에서 왔다.
> → 이 종목의 주가·이익 driver는 전사 매출이 아니라 **[실제 driver 세그먼트]의 [비중·성장·margin]**이다.

**테마/신성장 부문 실적 기여도 테스트** (관심 테마가 진짜 의미 있는지):

| 항목 | 현재 | 1~3년 후 | 판정 |
|---|--:|--:|:--:|
| 관련 매출 비중 |  |  | Optionality / Meaningful / Core |
| 관련 이익 비중 |  |  |  |
| 전사 매출 성장 기여 |  |  |  |

- 기준: 매출 비중 작아도 성장률·margin이 매우 높으면 Meaningful 이상일 수 있음. **반대로 기술적으로 중요해도 전사 기여 작으면 "Optionality(아직 작음)"라고 분명히.** sole vendor 여부보다 **전사 기여**가 중요.

## 8. 실적 driver: Q × P × Margin + 성장의 성격
실적을 Q(수량)·P(가격)·Margin으로 분해한다.
- **Q 증가 요인**: 고객 capex·신규 fab/DC/방산/전력 투자, 인증 완료, 증설, 수주잔고, 점유율, 교체/retrofit 수요.
- **P 상승 요인**: 공급 부족, 원재료 pass-through, 고부가 mix, 기술 난도 상승, switching cost, 가격 인상 근거.
- **Margin 요인**: 가동률, mix, 가격 인상, 비용효율, 신공장 ramp, 고정비 레버리지, 원가·공기지연 전가 가능성.

| Driver | 방향 | 강도 | 근거(데이터) | 지속성 |
|---|:--:|:--:|---|---|
| Q | ↑/→/↓ | Strong/Mid/Weak |  |  |
| P | ↑/→/↓ | Strong/Mid/Weak |  |  |
| Margin | ↑/→/↓ | Strong/Mid/Weak |  |  |

→ 한 줄 공식으로도 요약: `실적 = [Q변수] × [P변수] × margin`.

**성장의 성격** (delta가 진짜인지):

| 성장 driver | 구조적/순환적/일회성 | 실적 반영 시점 | 이미 보이나 vs narrative | 확인할 KPI |
|---|:--:|:--:|---|---|

- 이미 수주·실적·수출입·peer에서 보이는 성장인가, 아직 narrative 단계인가? 고객 capex timing과 수주 timing의 lag는?

**OP growth bridge (필수 산출) — 이익 성장의 출처를 세그먼트로 분해:**
§7의 성장 기여도%가 '매출' 성장의 출처라면, 여기선 '영업이익' 성장의 출처를 세그먼트로 쪼갠다. 매출을 끄는 세그먼트와 이익 성장을 끄는 세그먼트가 다를 때가 많다.
- **세그먼트 OP성장 기여% = (당기 세그먼트 OP − 전기 세그먼트 OP) ÷ 전사 ΔOP → 합 = 100%.**
- 과거 실적뿐 아니라 **forward 컨센 연도(향후 1~3년)에도 동일 적용**해, 전사 OP 성장을 좌우하는 **swing driver 세그먼트**를 특정한다.

| 세그먼트 | 전기 OP | 당기 OP | ΔOP | OP성장 기여% | (fwd) ΔOP | (fwd) 기여% | 신뢰도 |
|---|--:|--:|--:|--:|--:|--:|:--:|

> 전사 OP 성장의 X%는 [세그먼트]에서 나온다(매출 driver인 [A]와 다름). → 이 종목의 이익 driver = **[swing 세그먼트]**.

## 9. 시장은 이 종목을 어떻게 보는가 — 셀사이드 핵심 뷰 & 텐션
시장 프레임을 정리한다(리포트 문장 베끼지 말고 자기 말 요약 + 출처·날짜).
- **컨센 스냅샷**: 투자의견 분포, 목표주가 범위(최저~최고·중앙값), 현재가 대비 upside. *(숫자 인용일 뿐 우리 의견 아님.)*
- Bull / Bear / 텐션(같은 사실을 다르게 해석하는 논쟁) / 시장 주목 KPI·촉매 / 최근 rating·TP 변경.

| 구분 | 핵심 내용 | 근거 세그먼트·지표 | 출처·날짜 |
|---|---|---|---|
| Bull |  |  |  |
| Bear |  |  |  |
| 텐션 |  |  |  |

- **시장 통념 vs 실제(데이터)**: 시장이 가진 오해/단순화 1~3개와, §7~8 데이터가 보여주는 실제 구조를 대비. (verdict가 아니라 "데이터는 이렇게 말한다" 수준.)
- 셀사이드·컨센 없으면 "공개 커버리지 확인 안 됨" 명시.

## 10. 핵심 리스크 & tail risk
사업을 이해하려면 무엇이 깨질 수 있는지도 알아야 한다(thesis 판정은 하지 않음).

| Risk | 발생 가능성 | 충격 | 조기 경고 지표 |
|---|:--:|:--:|---|

- 핵심 리스크: 수요둔화·고객 capex 지연·가격하락·원재료/인건비·증설지연·dual sourcing·경쟁사 진입·환율·재고조정.
- tail risk(낮은확률·치명적): 기술 대체·주요고객 이탈·수출통제/규제·품질사고·정치/안보. 조기경고지표는 §4 인계로 ir-question-prep에 넘긴다.

## 11. 투자자가 가장 먼저 봐야 할 것
핵심 포인트 3~5개(통념과 다른 점·변동성·진입장벽·리스크 포함) + 앞으로 확인할 KPI/데이터 3~5개.
예(형식만): ①단순 소재가 아니라 기판 난도 상승 수혜 소모품 ②capex보다 생산량 민감 → 변동성 낮을 수 있음 ③매출은 범용이 크지만 이익 성장은 고부가가 주도 ④인증기간 길어 단기 진입 어려움 ⑤다만 dual sourcing 확대 시 ASP·점유율 압박.
