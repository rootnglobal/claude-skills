# Company Primer (초보자용 종목 심층 소개) 작성 스킬

> **공통 규칙**: 작업 시작 전 `_shared/common-rules.md`(로컬 실행 시 스킬 폴더 기준 `../_shared/common-rules.md`)를 읽고 전 구간에 적용한다 — fail-fast, 자동진행/사전확인, ASCII 파일명, 드라이브 3점 검증, 신뢰도 표기, 수집기간 기본 2년. 그리고 **태스크 완료 후 스킬 피드백 루프(§12)**: 최종 산출물 전달 직후(태스크 단위 1회) 새로 확인된 소스·URL·우회법, 반복된 실패 패턴, 사용자가 수동으로 고친 부분을 점검한다. 개선점이 있으면 업로드본 SKILL.md를 즉석 수정하고 동일 diff를 SSOT repo(claude-skills)에 반영한다 (repo 미접근 환경이면 diff를 출력해 "다음 Cowork 세션에서 repo 반영 필요"로 flagging). 없으면 "스킬 업데이트 없음" 한 줄로 종료한다.


리서처(RootN)용. 한 종목을 처음 보는 투자자가 **사업의 본질 / 고객이 돈을 내는 이유 /
실적이 움직이는 원리 / 시장이 보는 프레임**을 한 번에 이해하게 만드는, initiation report
앞단 같은 심층 소개 자료를 만든다.

**무게중심은 "회사에 대한 기초적 이해"다.** 산업구조·밸류체인·Q×P×Margin·moat·성장의 성격·
리스크까지 짚되, 그것은 모두 "이 회사가 어떻게/왜 돈을 버는지, 무엇이 진짜 driver인지"를
이해시키기 위한 것이다. 아래는 **이 스킬에서 하지 않는다**(다른 스킬의 몫):
- 투자의견·매력도 점수·BUY/Watch/Avoid·목표주가·밸류에이션 — **별도 투자판단 스킬**.
- 정량 추정·연간/분기 forward 모델·P×Q forecast — `investment-note`.
- 전체 peer 밸류에이션 비교·랭킹 — `comparative-investment-note`.
- IR 미팅 질문 리스트 — `ir-question-prep`. (단, 시장 텐션은 그쪽이 쓰도록 §4 인계만 한다.)
여기서는 **과거~현재 실적의 분해 + 산업/밸류체인 위치 + 시장 시각**으로 driver를 가려내는 데 집중한다.

## 0. 기본 설정 (묻지 말고 기본값으로 적용)
- **산출물: Word(.docx) 1개.** 서술형 글 + 표 다수이므로 docx가 맞다. 엑셀은 쓰지 않는다.
  작성 시 docx 스킬(`/mnt/skills/public/docx/SKILL.md`)을 읽고 그 컨벤션을 따른다(폰트 Malgun Gothic, 국문).
- **저장 위치 2곳:** ① `/mnt/user-data/outputs/`(ASCII 파일명) → `present_files`,
  ② 구글 드라이브 `Investment Research/[종목명 (티커)]/리서치/`에 업로드.
  → 드라이브에 올리면 미리보기·Google Doc 열기가 되고, 뒤따르는 `investment-note`·
  `ir-question-prep`가 `read_file_content`로 그대로 집어갈 수 있다(§4 인계 구조).
- **언어: 국문.**  **눈높이: 초보자.** 업계를 몰라도 한 번 읽고 "이래서 돈을 버는 회사구나"가 되게 쓴다.
- 데이터룸이 이미 있으면 그걸 읽고, 없으면 수집 스킬·웹·공시로 직접 채운다(§1).

## 1. 자료 수집 (research first — 출력 구조 채우기 전에 완료)

**파이프라인 위치:** 보통 IR·리서치 수집(`ir-collertor`/`jp-research-collector`/`news-collector`)
**다음에/함께** 쓰이고, 뒤에 `investment-note`·`ir-question-prep`가 따라온다. 데이터룸을 최대한
재사용하고, 산출물엔 뒤 스킬이 집어갈 **인계 메모**(§4)를 남긴다.

1. 종목 식별: 종목명/티커 → 거래소·코드·본국 언어 확정(한/영/일/중). 웹 1회.
2. **본국 언어 검색이 핵심.** 일본=일본어, 중국=중국어, 미국=영어로 회사·제품·고객·경쟁사를 검색한다.
3. 구글 드라이브 `search_files`로 'Investment Research/[종목명 (티커)]' 폴더·`fullText contains '종목명'`을 찾아
   IR·사업보고서·셀사이드·`news_digest_*`를 `read_file_content`로 정독한다.
   - **transcript 정독(있으면 필수):** `IR/transcripts/`의 어닝콜 transcript(CapIQ 수집분)는
     경영진이 사업을 **자기 말로 설명하는** 최고의 primer 소스다. 최근 2~4개를 정독해
     경영진이 반복 강조하는 KPI·성장 스토리·Q&A에서 애널리스트들이 파고드는 지점을 뽑는다
     (= §3-9 시장 프레임의 직접 입력). 없고 Chrome 가용이면 `capiq-collector`로 수집 시도.
   - 데이터룸이 비었으면 곧바로 `ir-collertor`(일본 `jp-research-collector`), 필요 시 `news-collector`를
     먼저 불러 1차 수집한다. 묻지 말고 진행하되 **fail-fast**(소스당 1~2회, 막히면 다음). 없으면 공시·공식 IR로 직접.
4. **supply chain 관점 필수.** 고객/모회사/경쟁사/소재·부품·장비 협력사의 제품·IR·뉴스로
   "사슬 어디에 있고 누구 때문에 버는지"를 잡는다. 이게 §3-4 밸류체인·peer의 입력이다.
5. **세그먼트 숫자(§3-7 입력):** 공시(사업보고서/10-K/有価証券報告書의 부문·지역)·IR·셀사이드에서
   지역·제품·고객별 매출·이익·성장률을 최대한 끌어온다. 부문/고객 이익은 공시가 약함 → 추정 시 "신뢰도 낮음".
6. **셀사이드 뷰(§3-9 입력):** 드라이브·뉴스 리포트에서 투자의견·목표주가(분포), bull/bear 논리,
   최근 rating/TP 변경, 시장 주목 KPI를 뽑는다. 문장을 베끼지 말고 **자기 말로 요약 + 출처·날짜**.
   - BEst 덤프(`Model/*_bbg_*.xlsx`)가 있으면 **Recs 시트(브로커별 의견·TP)**로 의견 분포를
     정량화하고, Revisions 시트로 컨센 방향(상향/하향 추세)을 한 줄 요약한다.
7. **산업·리스크 입력:** 경쟁구도·진입장벽·규제/정책·고객 capex cycle, 그리고 수요둔화·dual sourcing·
   대체기술·수출통제 등 리스크 단서를 함께 모은다(§3-3, §3-10).

## 2. 작성 원칙 (전 구간 적용)
1. **전문용어엔 반드시 쉬운 설명 + 비유.** 예: "프로브카드 = 칩 출하 전 검사하는 초정밀 바늘판", "초순수 장비 = 공장의 혈관".
2. **"이 제품이 없으면 고객에게 무슨 문제?"** 렌즈 = 고객이 왜 돈을 내는가 + 왜 못 바꾸는가(moat)의 답.
3. **매출 큰 사업 ≠ 이익 큰 사업 ≠ 성장 끄는 사업.** 셋을 숫자로 분리해 보인다(§3-7).
4. **숫자로 driver를 판별한다.** Country·Product·Customer × 매출·이익·**성장 기여도**를 표로 분해.
   성장 기여도 = 세그먼트 Δ매출 ÷ 전사 Δ매출(합=100%). 최소 최근 2~3년치. forward 추정은 안 한다(investment-note 몫).
5. **실적을 Q×P×Margin으로 분해한다.** 각 driver에 방향(↑/→/↓)·강도·근거·지속성을 단다(§3-8). 한 줄 공식으로도 요약.
6. **이해를 돕는 분석 렌즈를 쓰되 verdict는 내지 않는다.** 5+2 Forces·밸류체인·moat·성장의 성격·리스크는
   "사업을 이해"하기 위한 도구다. 매력도 점수·매수/매도·목표주가는 적지 않는다.
7. **시장 프레임을 보여준다.** 셀사이드 bull/bear/텐션 + "시장 통념 vs 데이터가 보여주는 것". 투자자 입장의 출발점.
8. **모든 수치·주장에 출처.** 추정·저신뢰는 "신뢰도 낮음"(별/반/세모) 표기. 모르면 "확인 필요". 지어내지 않는다.

## 3. 출력 구조 (반드시 이 골격을 따른다)
각 섹션의 세부 질문·표 양식은 **`references/output-template.md`를 읽고 그대로** 따른다.

1. **회사 한 줄 정의** — 무엇을 만들어 어떻게(반복적으로) 버는지 한 문장.
2. **무엇을 만드는 회사인가** — 핵심 제품/서비스 + 공정 내 위치 + 쉬운 비유 + 교체주기.
3. **산업 구조 (5+2 Forces)** — 5 Forces + (규제/정책/안보, capex cycle/고객 투자 timing). 각 항목이 이 회사에 유·불리한지 + 산업 매력도 표.
4. **밸류체인 위치 & 경쟁구도** — upstream/mid/down 표(단계·주요기업·병목·가격결정력·이 회사 노출) + 주요 peer와 이 회사의 차별점(상세 비교는 comparative-investment-note로 위임) + 인증기간·dual sourcing 가능성.
5. **고객은 왜 이 회사 제품을 사는가 + Moat** — pain point, 품질/수율/납기/인증 중 결정 요인, 원가 비중 vs 고장 손실, 전환비용. 이어서 "돈으로 연결되는 moat" 표(인증·lock-in·capacity·가격결정력).
6. **돈을 버는 방식** — 6개 유형 분류 + capex/생산량 연동 + 수주→매출 lag + recurring 비중 + pass-through·FX 민감도.
7. **세그먼트 Breakdown (숫자)** — Country/Product/Customer × 매출·이익·성장기여 표 + driver 판별 1단락. 이어서 테마/신성장 부문의 실적 기여도 테스트(Optionality / Meaningful / Core Driver).
8. **실적 driver: Q×P×Margin + 성장의 성격** — Q·P·Margin 각각 방향·강도·근거·지속성 표 + 한 줄 공식. 성장을 구조적/순환적/일회성으로 분류하고 "이미 실적에 보이나 vs 아직 narrative인가".
9. **시장은 이 종목을 어떻게 보는가** — 셀사이드 컨센·목표주가, Bull/Bear/텐션 표, 시장 주목 KPI + "시장 통념 vs 실제(데이터)".
10. **핵심 리스크 & tail risk** — 주요 리스크와 낮은확률·치명적 tail risk를 표로(발생가능성·충격·조기경고지표). 사업 이해 관점(thesis 판정 아님).
11. **투자자가 가장 먼저 봐야 할 것** — 핵심 포인트 3~5개(통념과 다른 점·변동성·진입장벽 포함) + 앞으로 확인할 KPI/데이터 3~5개.

> 문서 맨 위에 **요약 박스**(한 줄 정의 + Q×P×Margin 한 줄 + 핵심 포인트 3~5개)로 "이것만 봐도 절반은 이해"되게.

## 4. 산출 및 전달 (+ 다음 스킬 인계)
- docx 스킬을 읽고 Word 1개로 작성한다(본문 구조 §3 + 템플릿). 표(Forces·밸류체인·세그먼트·Q×P×M·셀사이드·리스크)는 docx 표로.
- 저장: `/mnt/user-data/outputs/`(예 `company_primer_<ticker>.docx`) → `present_files`, + 드라이브 종목 폴더 `리서치/` 업로드.
- **맨 끝에 "다음 스킬용 인계 메모"를 반드시 넣는다:**
  - *investment-note용 정량 단서*: **§7 세그먼트 breakdown 표 = 모델 시작점.** + Q×P×Margin driver, P×Q 분해 후보(단가·물량 힌트), capex/생산량 연동, 고마진·저마진, 가동률·mix 레버, 성장의 성격. 저신뢰 태그 유지.
  - *ir-question-prep용 텐션 포인트*: **§9 셀사이드 bull/bear 텐션 + §10 리스크의 조기경고지표** + 공개자료로 확인 안 된 가정 + 경영진만 답할 수 있는 미해결 질문 후보(거친 형태라도). JP/CN은 현지어 핵심 표현 병기.
  - 1차 목적은 사람 읽기가 아니라 **뒤 스킬의 입력**이다.

## 5. 검증 및 원칙
- 비유는 **정확해야** 한다(멋지지만 틀린 비유보다 투박해도 맞는 비유).
- sole vendor라 단정하지 말고 multi-vendor·dual sourcing·ASP 압박 가능성을 점검한다.
- 기술적으로 중요해도 **전사 실적 기여가 작으면 작다고 분명히** 적는다(§3-7 테마 기여도 테스트).
- 투자의견·점수·목표주가·밸류에이션·IR 질문 리스트는 넣지 않는다(다른 스킬의 몫). 내부 참고용 면책 한 줄을 끝에.

## 기술 주의 (산출 안정성)
- 파일명·멀티바이트 기호 처리는 공통 규칙 §3을 따른다(docx는 heredoc 생성).
