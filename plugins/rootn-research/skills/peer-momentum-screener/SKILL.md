# Peer Momentum Screener

기준 종목(anchor) 대비 피어들의 상대 매력을 **선행지표 delta** 중심으로 판정하는 스크리닝 워크플로우.
핵심 사상: 레벨(얼마나 좋은가)이 아니라 **가속(좋아지는 속도가 빨라지는가)**으로 종목을 고른다.

## Step 0. 유니버스 확정

- 시작 전 `_shared/common-rules.md`의 공통 규칙(fail-fast, 신뢰도 표기, 파일명 ASCII 등)을 따른다.
- **태스크 완료 후 스킬 피드백 루프(§12)**: 최종 산출물 전달 직후(태스크 단위 1회) 새로 확인된 소스·URL·우회법, 반복된 실패 패턴, 사용자가 수동으로 고친 부분을 점검한다. 개선점이 있으면 업로드본 SKILL.md를 즉석 수정하고 동일 diff를 SSOT repo(claude-skills)에 반영한다 (repo 미접근 환경이면 diff를 출력해 "다음 Cowork 세션에서 repo 반영 필요"로 flagging). 없으면 "스킬 업데이트 없음" 한 줄로 종료한다.
- 기준 종목 1개 + 사용자 지정 피어. 사용자가 "다른 수혜주도"라고 하면 밸류체인 확장 후보(전방·후방·인접 공정)를 제시하되 진행을 멈추지 말 것.
- 결과가 갈리는 순수 식별 질문(동명 종목·티커 모호)만 확인 질문 허용. 그 외에는 합리적 기본값으로 진행 후 가정을 명시.

## Step 1. 종목별 핵심 선행지표(Key Leading KPI) 선정 ★출발점

종목마다 "매출·이익을 2~6분기 선행하고, 회사(또는 업계 통계)가 분기 이상 빈도로 일관 공시하는 지표"를 하나씩 지정하고 선정 근거를 시트에 명기한다.

섹터별 기본값(예시 — 이에 갇히지 말 것):

| 섹터 | 기본 KPI | 미공시 시 프록시 |
|---|---|---|
| 건설·플랜트·설비공사 | 분기 net 수주, 잔고 커버리지 | 착공 통계, 발주 파이프라인 |
| 산업기계·장비·SPE | 수주 / Book-to-Bill | 고객 capex 가이던스, WFE·업계 출하 통계, 리드타임 코멘트 |
| 소재·화학 | 출하량(Q)·가동률·스프레드 | 업계 재고순환(재고↓+출하↑ 국면 여부) |
| 전자부품 | 수주잔고, 월차 매출 | 주요 고객 빌드 계획 |
| 소비재·리테일 | 기존점(SSS) 월차, 객수×객단가 | 카드 데이터·업계 판매 통계 |
| SaaS·인터넷 | ARR·NRR·과금계정, EC=GMV | 앱 다운로드·트래픽 |
| 금융 | 은행=대출 증가율·NIM 방향, 보험=신계약 ANP·CSM 증가 | — |
| 제약·바이오 | 파이프라인 마일스톤 + 주력품 처방 데이터 | — |
| 운송·리스·인력 | 물동량·계약잔고·가동인원×단가 | — |

- **미공시 종목 처리**: (a) 프록시를 쓰되 출처·산출법·"신뢰도 낮음" 표기, (b) 프록시조차 없으면 가이던스 개정 이력 + 컨센서스 리비전 방향으로 판정하고 판정 열에 그 사실을 명시. 지표가 없다고 종목을 빼지 말 것.
- **anchor와 peer의 지표가 서로 달라도 됨**: 각자 자기 지표의 delta로 판정하고, 횡비교는 표준화 축(분기 YoY, TTM YoY, 가속도 Δ%p)으로만 한다. 절대 레벨을 지표 간 직접 비교하지 않는다.

## Step 4. 시트 1 "비교스크리닝"

- 열: 구분 / 종목(코드) / FY실적 매출·OP·OP YoY·OPM(수식) / 가이던스·YoY(수식) / 셀사이드 추정 / 핵심 선행 KPI(종목별 지표명 명기)·KPI YoY / 스톡 지표(잔고·ARR 등)·커버리지(수식) / PER / PBR / 배당수익률 / 테마 노출(정규화 %, 아래 규칙) / 환원 정책 / 판정(◎○△✕) / 신뢰도·비고.
- 기준 종목 행 최상단 음영. 하단 각주: 판정 기준, 결론 요약, 신뢰도 낮음 목록, 미확보/실패 목록.
- xlsx 스킬 규칙 준수: 하드코딩 파랑·수식 검정, 계산은 Excel 수식, recalc로 에러 0 검증.

### 테마 노출 정규화 규칙 (필수 — 절대치 단독 금지)

회사마다 규모가 다르므로 "탱커 N척" 같은 절대치만으로는 thesis 민감도를 비교할 수 없다.
테마 노출은 반드시 **서로 비교 가능한 %**로 제시하고, 절대치는 그 산출 근거로만 병기한다:

1. **자산 비중%** = 테마 자산 ÷ 전사 동종 자산 (예: 탱커 척수/총 선대, 테마 캐파/총 캐파). 가능하면 dwt·㎡ 등 용량 기준도 병기.
2. **매출 비중%** = 테마 세그먼트 매출 ÷ 전사 매출 (FY1E 기준, 미공시면 FY-1A 대용 후 명기).
3. **이익 비중%** = 테마 세그먼트 이익 ÷ 전사 핵심이익 — 주가 민감도의 1차 근사. 세그 이익 미공시면 가이던스 증감요인 금액을 스윙%로 대체.
4. **단위 민감도%** = 테마 가격변수 1단위 변동(예: VLCC $10k/day, 메모리 $1, 운임 100pt)당 연 순이익 영향 ÷ 순이익. 회사 공시 감도 최우선, 없으면 back-of-napkin(자산 수 × 가동일 × 스팟 비중)으로 추정하고 "신뢰도 낮음" 표기.
5. **계약구조 레이어 (스팟 vs 장기)** = 테마 자산 중 실제로 시황에 열려 있는 비중. 자산이 있어도 장기계약(COA·기간용선·CVC 등)이 부착되어 있으면 테마 수혜는 0에 수렴한다(예: 척수 비중 36%인데 원유·가스 스팟 0%면 VLCC 급등 수혜 없음). **회사 공시 익스포저(시황:안정 비율, 익스포저 척수/일수) 최우선** — 일본 종목은 결산설명자료 보충자료·中期経営計画을 현지어로 검색(예: "市況エクスポージャー", "安定利益 比率"). 미공시면 추정 후 신뢰도낮음.
- **개수 % 함정**: 척수·건수 등 개수 기준 %는 자산 크기(dwt·캐파)와 시장(동일 "탱커"라도 VLCC와 케미컬은 별개 운임)과 계약 부착을 모두 무시한다 — 반드시 용량 기준·계약구조 레이어와 병기하고, 개수 % 단독으로 판정하지 않는다.
- **노출 크기보다 구성**: 노출 %가 비슷해도 최대 이익원이 테마와 무관하거나 역행(드래그 자산)하면 판정이 갈린다. 이익 비중 상위 1~2개 사업이 테마의 순풍/무풍/역풍 중 어디인지 한 줄로 명기(예: 자동차선 45%가 연간계약 고정+물동량 역노출이면 드래그).
- **다축 테마는 축별로 분리**: 테마가 여러 가격축(예: VLCC vs SCFI)이면 노출표를 축별로 따로 만든다 — 같은 회사가 축에 따라 순위가 뒤집힌다.
- % 셀은 전부 수식(분자·분모를 파랑 입력 셀로 분리). pure play는 100%로 명기해 스케일 차이를 드러낸다.
- 해석 각주 필수: "%가 높다 = 민감하다"이지 "좋다"가 아니다(하락 국면 역행). anchor 대비 배율(예: "DHT 민감도는 MOL의 2.3배")을 한 줄로 명기.
- 전용 시트("노출도정규화")로 분리해도 되고, 비교스크리닝 열에 직접 넣어도 된다 — 어느 쪽이든 절대치 단독 표기는 금지.

## Step 5. 시트 2 "선행지표모멘텀" ★핵심

- 종목별 KPI 시계열 8~12개 분기(월차 지표면 24~36개월). 헤더에 종목별 지표명·기준(연결/단체, 회사/업계 통계) 명기.
- 누적 공시(일본 분기 수주 등)는 차분으로 net 산출. YoY 시작점 확보 위해 표시 구간보다 1년 앞선 데이터도 수집(직전년도 공시의 전년동기 비교란에서 취득 — 보간·추정 금지).
- 기준 통일: 종목당 하나의 일관 기준만, 혼용 금지. 각 연도 수치를 이듬해 공시의 전년비교란과 크로스체크.
- 블록: ① 분기(월차) KPI ② YoY(수식) ③ TTM 합 또는 이동평균(수식, 데이터 부족 시 공란) ④ TTM YoY(수식) ⑤ 가속 판정 = [최근 2Q 평균 YoY − 직전 4Q 평균 YoY] + [TTM YoY 기울기(최근−4Q전)], 둘 다 +5%p 초과=가속 / 한쪽 −5%p 미만=감속 / 그 외 중립 ⑥ 회사 차기 KPI 계획/가이던스 YoY 행 ⑦ TTM YoY 라인차트.
- 예외: 결산기·기준 변경으로 연속 비교 불가 종목(예: Daifuku 3월→12월 결산 변경)은 매트릭스에서 빼고 별도 블록에 클린 비교 구간만 명기. 프록시 지표 종목도 별도 블록.
- 각주에 기계식 판정의 한계 명시: lumpy 종목(대형 단건·계절성)은 분기 YoY 노이즈로 오판 가능 → TTM 추세선과 병행 판독. 월차 지표는 3개월 이동평균 우선.

## Step 6. 판정·결론 (검증된 원칙)

- **헤드라인 KPI 가속 ≠ 테마 가속**: 반드시 내역·믹스로 순도 확인(회사의 구분 정의 원문 확인. 예: Dai-Dan 총수주 +25% 가속이었지만 DC·반도체가 포함된 '산업시설' 수주는 −20%, 견인차는 해외 M&A·리뉴얼).
- **one-off vs 지속 가능 분해**: M&A 연결 편입(첫해 YoY만 부풀림)·대형 단건·특별이익·환율 vs 캐파 연동 성장(인력·설비 × 단위 생산성)·리뉴얼/서비스/구독 스톡. 증가분을 금액 단위로 쪼개 귀속.
- **이익의 질**: 마진 개선의 원천(가격전가/믹스/일회성 정산/환율) 구분.
- **자산·노출 수치의 최신성 검증**: 선대·캐파 등 노출의 분자가 되는 수치는 M&A·연결범위 변경으로 급변한다(예: NEO 연결로 탱커 68→106척). 셀사이드 리포트의 수치를 쓰기 전에 최신 결산단신·설명자료의 선대표와 대조하고, 6개월 이상 지난 수치는 기준일을 명기한다.
- **시황 방향 단정 금지 (사전 검증 의무)**: 판정 근거로 시황(운임·가격·수급)의 방향을 쓰려면, 쓰기 전에 반드시 스크리닝 시점의 최신 스팟 데이터(주간 지수 레벨, 연속 상승/하락 주수, 컨센서스 리비전 방향)를 검색·확인한다. **'구조 전망'(공급 사이클, 수년)과 '현재 delta'(스팟 방향, 주·월)는 별도 축으로 분리 표기**하고, 구조 프레임만으로 "최약체/downcycle" 같은 단정을 내리지 않는다. 두 축이 상충하면(예: 구조 공급과잉 vs 스팟 18주 연속 상승) 상충 사실 자체를 판정 근거에 명시한다.
- 판정 변경 시 사유를 비고에 기록(예: "KPI 감속 확인으로 ◎→○ 하향").
- 다음 검증 이벤트(각 종목의 차기 KPI 발표일)를 결론에 명시하고 갱신 스케줄 제안.

## 행동 규칙

- 자료 수집·파일 생성은 묻지 않고 진행. 판단이 갈리는 지점(유니버스 범위, KPI 선정이 애매한 종목, 풀 모델 확장 여부)은 명시적으로 짚어 사용자가 선택.
- 파일명 ASCII + 날짜, 재생성 시 _v2. 다바이트 포함 스크립트는 heredoc bash.
- 최종 응답은 랭킹과 "기준 종목 대비 무엇이 다른가"를 숫자로 압축. 챗 결론과 Excel 각주를 일치.

## 기존 스킬과의 관계

- 자료 수집 부족 시 jp-research-collector / ir-collertor / fnguide-report-downloader 호출(재구현 금지).
- 특정 종목 심화 요청 시 company-primer(이니시에이션) 또는 comparative-investment-note(풀 모델)로 연결 제안.
- 산출물 규격은 xlsx 스킬 표준.
