# 비교 Investment Note 스킬 (Comparative Investment Note)

> **공통 규칙**: 작업 시작 전 `_shared/common-rules.md`(로컬 실행 시 스킬 폴더 기준 `../_shared/common-rules.md`)를 읽고 전 구간에 적용한다 — fail-fast, 자동진행/사전확인, ASCII 파일명, 드라이브 3점 검증, 신뢰도 표기, 수집기간 기본 2년, 그리고 **판정 근거 검증 원칙(§11)**: 노출 정규화 %(절대치 단독 금지·개수% 함정)·계약구조 레이어(스팟 vs 장기)·노출 크기보다 구성(드래그 자산)·다축 분리·시황 방향 사전 검증(구조 vs 현재 delta)·수치 최신성. 그리고 **태스크 완료 후 스킬 피드백 루프(§12)**: 최종 산출물 전달 직후(태스크 단위 1회) 새로 확인된 소스·URL·우회법, 반복된 실패 패턴, 사용자가 수동으로 고친 부분을 점검한다. 개선점이 있으면 업로드본 SKILL.md를 즉석 수정하고 동일 diff를 SSOT repo(claude-skills)에 반영한다 (repo 미접근 환경이면 diff를 출력해 "다음 Cowork 세션에서 repo 반영 필요"로 flagging). 없으면 "스킬 업데이트 없음" 한 줄로 종료한다.


RootN 리서처용. **2개 이상의 종목을 받아 하나의 Excel에서 비교 + 종목별 모델까지
끝내는 오케스트레이터**다. 단일 종목 노트는 `investment-note`가, 단일 종목 풀
리서치는 `ir-research-pipeline`이 담당한다. 이 스킬은 그 위에 **비교(comparison)
레이어**를 얹는다.

이 스킬의 고유 가치 = **종목들을 진짜로 '비교 가능하게' 만드는 것**이다:
- 항목·단위·회계기준·통화·결산월을 종목 간에 정규화한다.
- 바스켓 전체에 공통으로 작용하는 **매크로 동인(예: 에너지=유가·가스가·환율,
  유틸리티=전력단가·연료비·원전 재가동)**을 정의하고, **각 종목의 그 동인에 대한
  민감도(상대 레버리지)**를 같은 틀로 보여준다.
- 비교 시트의 모든 숫자는 종목별 모델 시트에서 **수식으로 끌어와** 한 곳에서 본다.

## 호출하는 전용 스킬 (building blocks — 직접 다시 만들지 말 것)

| 단계 | 사용하는 스킬 | 핵심 역할 |
|---|---|---|
| 종목별 풀 수집 | `ir-research-pipeline` | 종목 1개의 IR·셀사이드·Gmail 수집~정리를 통째로 위임 |
| IR·공시 수집 | `ir-collertor` | 공식 IR·공시(DART/SEC/EDINET/CNINFO)를 드라이브 `IR/`에 |
| JP 리서치 | `jp-research-collector` | (일본) Kabutan·Minkabu·Kabuyoho 컨센서스 + IR PDF + Shared Research |
| 셀사이드(한국) | `fnguide-report-downloader` | FnGuide 애널리스트 리포트 PDF |
| CapIQ 수집 | `capiq-collector` | (Chrome 가용 시) transcripts·미중 셀사이드·컨센 상세 export |
| BBG 인제스트 | `bbg-sync` | (브리지 가용 시) Bloomberg_Inbox 덤프·문서를 종목별 Drive로 |
| 종목별 모델 | `investment-note` | 종목 1개의 Excel 모델(1p 요약 + 연간·분기, 세그먼트 P×Q, 밸류) |
| 드라이브 정리 | `drive-folder-organizer` | 종목 폴더를 IR/·리서치/로 분류, 중복 제거, 폴더명 표준화 |
| 산출 포맷 | `xlsx` | 비교 Excel 작성(openpyxl), recalc 검증 |

> 전용 스킬을 부를 때는 **그 SKILL.md를 읽고(또는 Skill 도구로 불러) 절차·도구·파일명
> 규칙을 그대로 따른다.** 이 문서는 바스켓 흐름, 정규화 규칙, 비교 시트 설계만 정의한다.

## 기본 설정 (묻지 말고 기본값으로)
- **산출물: Excel 1개.** 첫 시트 '비교요약' = N종목 side-by-side, 이후 종목별 모델 시트.
- **언어: 국문.**
- **시계열:** 종목별 모델은 연간·분기 모두 최근 3년 + 향후 3년(가능하면 더 길게),
  `investment-note` 규칙을 따른다.
- (사용자가 비교 전용·경량 또는 Word 비교 노트를 명시 요청하면 그에 맞춘다.)

## Phase 0. 바스켓 정의 & 사전 점검

1. **바스켓·종목 식별**: 각 종목명/티커 -> WebSearch로 정식명(현지어+영문)·코드·거래소·
   국가·결산월·보고통화를 확인한다(종목당 1회). 일본 4자리=일본, 6자리/한글=한국,
   영문 티커=미국, 그 외=중국/확인.
2. **비교 축(comparison axes) 확정**: 바스켓 성격에 맞는 공통 비교 항목을 정한다.
   - 공통: 밸류(PER/PBR/EV·EBITDA/배당·FCF수익률), 실적(매출/OP/OPM/순익/ROE),
     성장(매출·EPS CAGR), 재무건전성(순차입금/EBITDA, 이자보상).
   - **테마 고유 매크로 동인**을 1~3개 정의(아래 매크로 동인 라이브러리 참고).
3. **결산월·통화·회계기준 정규화 방침**: 결산월이 다르면 '직전 4분기 합(LTM)' 또는
   동일 역년 기준으로 맞춘다. 통화가 다르면 비교표에 **공통통화 환산행**을 둔다.
   회계기준(IFRS/US GAAP/J-GAAP) 차이가 큰 항목(예: 영업이익 정의)은 주석으로 표기.
4. **기존 드라이브 자산 점검**: 각 종목의 `Investment Research/...` 폴더 존재 여부를
   `search_files`로 확인. 있으면 재사용, 없으면 수집 단계에서 생성. 테마 폴더
   (예: '일본 에너지')가 지정됐는지도 확인한다.
5. **Bloomberg 자산 자동 점검(묻기 전에 먼저)**: 각 종목 `Model/`의 BBG 템플릿·덤프
   (`*_bbg_*.xlsx`)를 확인하고, 브리지가 살아 있으면 `bbg-sync`로 바스켓 전 종목의
   인박스 신규 파일을 한 번에 인제스트한다(전체 모드). 컨센 위계는 종목 공통으로
   BEst 덤프 > CapIQ export > 무료 소스 — **비교표는 같은 위계의 소스로 통일**하고,
   섞이면 종목별 소스를 각주로 표기한다.
6. **추가 자료 1회 확인(AskUserQuestion)**: 본격 작업 전 한 번만 - (자동 점검에서 못
   찾은) Bloomberg 템플릿/컨센서스 엑스포트 유무, 비교 깊이(풀 모델 vs 비교전용),
   드라이브 정리 동의.

작업 시작 시 TaskCreate로 (1) 종목별 수집 (2) 드라이브 정리 (3) 비교 Excel (4) 검증
작업을 만들어 진행 상황을 보여준다.

## Phase 1. 종목별 자료 수집 (research first - 종목마다 반복)

각 종목에 대해 **개별적으로** `ir-research-pipeline`의 Phase 1(수집)을 수행하거나,
일본 종목이면 `jp-research-collector` + `ir-collertor`를 직접 부른다.

- **병렬화**: 종목이 3개 이상이고 수집이 독립적이면, 종목별 수집을 **subagent(Task 도구)로
  병렬** 실행해 시간을 줄인다. 각 subagent에는 "이 종목의 IR·셀사이드·컨센서스를 모아
  핵심 수치(매출/OP/순익/EPS/세그먼트/단가·물량/컨센서스/매크로 민감도)를 표로 반환"
  하도록 지시한다.
- 수집물의 최종 목적지는 드라이브 `Investment Research/[종목명] (티커)/` (또는 지정 테마 폴더).
- **비교를 위해 종목 간 동일 항목을 반드시 확보**한다: 같은 회계연도들의 매출·OP·순익·
  EPS·BPS·DPS, 세그먼트 매출구성, 그리고 **공통 매크로 동인에 대한 민감도**(예: 유가
  $1/bbl 변동 -> 경상이익 ΔX, 전력단가 1원/kWh -> ΔY). 민감도가 공시·리포트에 없으면
  back-of-napkin으로 추정하고 "신뢰도 낮음" 표기.
- 컨센서스(향후 2~3년)는 종목별로 텍스트 정리 -> Phase 3 비교표에서 재사용.

> `investment-note`의 데이터 원칙(저신뢰 수치도 버리지 않고 표기, 세그먼트 최대 세분화,
> P×Q 분해 필수)을 종목마다 동일하게 적용해야 비교가 공정해진다.

## Phase 3. 비교 Excel 작성 (`xlsx` 스킬 -> openpyxl)

### 3-1. 시트 구성
```
비교요약            <- N종목 side-by-side (이 스킬의 핵심, 3-2)
매크로_민감도        <- 공통 동인에 대한 종목별 민감도·시나리오
[종목A] 모델         <- investment-note 구조(요약·연간·분기·세그먼트·P×Q·밸류)
[종목B] 모델
[종목C] 모델
가정·출처
```
- 종목별 모델 시트는 `investment-note` 8-6 구조를 1시트로 압축하거나, 종목이 적으면
  종목별로 여러 시트를 둔다. **비교요약의 모든 셀은 이 모델 시트를 수식으로 참조**한다
  (하드코딩 금지 -> 모델 수정 시 비교표 자동 갱신).

### 3-2. '비교요약' 시트 표준 행 (종목=열, 항목=행)
같은 단위·통화로 정규화한 뒤 다음 블록을 세로로 쌓는다:
- **스냅샷**: 현재가, 시가총액, 순차입금, EV, 발행주식수, 결산월, 보고통화.
- **밸류에이션**(현재 + 향후 1~3년): PER, PBR, EV/EBITDA, EV/EBIT, 배당수익률, FCF수익률.
- **실적**(최근 3년 + 향후 3년): 매출, 영업이익(또는 경상이익), OPM, 지배순익, EPS.
- **성장**: 매출 CAGR, EPS CAGR, 향후 컨센 성장률(종목별 컨센서스 인용).
- **수익성·재무**: ROE, ROIC, 순차입금/EBITDA, 이자보상배율, 배당성향.
- **세그먼트 구성**: 매출 mix(%)와 '성장의 출처'(어느 세그먼트가 성장을 끄는지 키워드).
- **매크로 레버리지**: 공통 동인 1단위 변화당 이익 민감도(절대·% 둘 다) -> 3-3.
- **투자뷰**: 종목별 Bull/Bear 한 줄 + 상대 랭킹(밸류 매력·성장·레버리지 축별).
- 음수 괄호, 0은 "-", % 0.0%, 배수 0.0x. 저신뢰 셀은 "신뢰도 낮음" 표기.
- **상대 비교 보조**: 항목별로 최저PER·최고성장 등을 조건부서식/랭크 열로 강조.

### 3-3. 매크로_민감도 시트 (비교의 백본)
- 바스켓 공통 동인(예: 유가, 가스가, USD/JPY, 전력 스프레드)을 행으로 두고,
  각 종목의 **이익 민감도**(공시·리포트 인용 또는 back-of-napkin 추정)를 열로 둔다.
- 동인 시나리오(Bear/Base/Bull)별로 종목별 경상이익·EPS·이론주가 변화를 산출 ->
  "같은 매크로 환경에서 어느 종목이 가장 크게 움직이나"를 한눈에 보이게.
- 추정 민감도는 가정(생산량·판매단가 연동·헤지·규제 pass-through)을 입력 셀로 명시하고
  출처·신뢰도를 단다.

### 3-4. 작성 규칙(investment-note 준용)
- 단위 통일(억엔/억원 등), 폰트 Arial, 색코딩(파랑=입력, 검정=수식, 초록=시트참조).
- 비교요약<->모델 시트 링크, 분기 합=연간 일치, P×Q 재계산.

## 매크로 동인 라이브러리 (테마별 공통 동인 예시)
- **에너지(E&P/정유/가스)**: 유가(Brent/WTI $/bbl), 천연가스/LNG가, 정제마진, USD/JPY·환율,
  생산량(boe/d), 매장량·매장량수명(R/P). E&P는 가격 레버리지, 정유는 마진 레버리지.
- **전력·유틸리티**: 전력 도매단가, 연료비(LNG·석탄), 연료비조정요금 pass-through,
  원전 재가동·가동률, 송배전 규제자산(RAB)·허용수익률, 금리(요금기저 영향).
- **반도체·IT**: 메모리 가격, 가동률, 환율, 설비투자 사이클, 고객 CAPEX.
- **소재·화학**: 원료(납사·메탈) 가격, 스프레드, 가동률, 수출물량.
- **금융·은행**: 기준금리·NIM, 대손율, 자본비율.
- 바스켓이 이 라이브러리에 없으면, 매출·원가 구조를 보고 **가격 1개 + 물량 1개 + 환율**을
  기본 공통 동인으로 잡는다.

## 원칙
- **비교 가능성이 최우선**: 같은 단위·통화·기간·회계기준으로 맞춘 뒤에만 숫자를 나란히 둔다.
- **전용 스킬 신뢰**: 종목별 수집·모델은 검증된 스킬에 위임하고, 이 스킬은 정규화·비교·랭킹·
  매크로 민감도에 집중한다.
- **수식으로 연결**: 비교표는 모델 시트를 참조해 자동 갱신되게 한다(하드카피 금지).
- **저신뢰도 보존·표기**, **back-of-napkin(P×Q·민감도) 직접 수행**, **출처 주석**.
- 파괴적 작업(드라이브 이동·메일)은 사용자 확인 후. 내부 참고용 면책 포함.
